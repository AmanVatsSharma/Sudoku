# Contributing to Ultimatum Sudoku

Thank you for taking the time to contribute. This project is open source under the [MIT License](LICENSE).

## Ways to help

- **Report issues** — Bug reports, confusing UI, or suggestions belong in [GitHub Issues](https://github.com/AmanVatsSharma/Sudoku/issues). Include steps to reproduce, device/OS when relevant, and screenshots if they help.
- **Submit pull requests** — Small, focused PRs are easier to review than large refactors. One concern per PR (for example: fix a bug, add a test, or improve copy—not all at once).
- **Improve docs** — README clarifications, build troubleshooting, or translations are welcome.
- **Tests** — New behavior should include or update tests when practical (`__tests__/`).

## Before you open a PR

1. Install dependencies: `npm install`
2. Run quality checks:

   ```bash
   npm run lint
   npm run typecheck
   npm run format:check
   CI=true npm test
   ```

3. Run `npx expo-doctor` if you changed native config, Expo version, or dependencies.

## Code expectations

- Match existing **TypeScript**, **formatting** (Prettier), and **patterns** in [`src/`](src/).
- Prefer extending existing modules over duplicating logic.
- Do not commit **secrets** (keystores, API keys, `.env` with private data).
- The [`android/`](android/) and [`ios/`](ios/) trees are generated by Expo prebuild and are not checked in; keep changes in `app.config.ts` and JS/TS unless you have a strong reason to document native steps in the README.

## Review process

Maintainers will review PRs when they can. Be ready to clarify intent or make small follow-up changes. If a PR sits idle, a polite ping in the thread is fine.

Thank you for helping make Ultimatum Sudoku better.
